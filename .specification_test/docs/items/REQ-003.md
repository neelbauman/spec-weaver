# [REQ-003] 生きた仕様書 (Living Documentation)

### 内容

## 概要
エンジニアとビジネスサイドが共通して読める、美しく検索可能な
静的ドキュメントサイトを自動生成すること。

## 背景・動機
YAMLや.featureファイルは開発者向けのフォーマットであり、ビジネスサイドのステークホルダーが
直接閲覧するには不適切である。仕様と実装状況を統合した、ナビゲーション可能な
ドキュメントサイトを自動生成することで、「生きた仕様書」を実現し、
チーム全体での仕様理解を促進する。

## 受け入れ基準
- Doorstopの要件・仕様データからMkDocs用のMarkdownを自動生成できること
- 要件一覧・仕様一覧のテーブルが生成されること
- 個別アイテムの詳細ページが生成されること
- REQ-SPEC間の相互リンクが含まれること
- テストシナリオのファイルパス・行番号が表示されること
