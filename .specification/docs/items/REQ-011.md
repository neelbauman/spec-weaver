### 内容

## 概要
`.feature` ファイルからpythonの behave 互換のテストコード雛形を自動生成し、
CI/CD パイプラインでのテスト実行・結果反映を一気通貫で行えるようにする。

## 背景・動機
現在、`.feature` ファイルは仕様書としてのみ機能しており、
behave による自動テストとの接続は手動で行う必要がある。
`scaffold` コマンドで BDD テストコードの雛形を自動生成し、
テスト実行結果を `build` ドキュメントに反映するまでのフローを
自動化することで、仕様→テスト→ドキュメントのトレーサビリティを強化する。

## 受け入れ基準
- `.feature` ファイルから behave テストコードの雛形を自動生成できること
- 日本語のシナリオ名・ステップ名が関数名に混入しないこと（ハッシュベースの命名）
- 生成されたテストを実行し、Cucumber 互換 JSON を出力できること
- CI/CD パイプラインで scaffold → テスト実行 → build の一連フローが実行可能であること
